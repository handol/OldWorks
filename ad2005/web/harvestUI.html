테스트: URL  


l          기본적으로는 주어진 URL (root URL)의 문서 하나만 가져 온다.

l          Root URL의 문서 내에 포함된 link들의 문서를 가져오기 위해서는 ‘-D’ d옵션을 사용한다.

l          –D 숫자 : 주어진 숫자는 depth를 의미한다. Link는 몇 depth까지 따라갈 것인지 지정하는 것이다.

l          –m 숫자: 하나의 웹 문서에서 추출하는 하위 링크의 최대 개수

l          –M 숫자: 한번의 문서 수집 실행에서 최대로 수집할 수 있는 문서의 최대 개수.

l          –I 디렉토리목록: 기본적으로는 root URL에 주어진 path 아래의 url주소만 따라간다. 예을들어 www.cnn.com/ASIA 인 경우, /ASIA 하위의 디렉토리만 link를 따라간다. 그외 디렉토리를 포함하고자 한다면 ‘-I’로 지정을 하여야 한다.

Ø         Include의 의미.

Ø         디렉토리목록은  코마 기호(‘,’)로 나열한다.

Ø         예) -I /2005/WORLD,/WORLD

l          -X 디렉토리목록: 특정 디렉토리를 제외(Exclude)하기 위한 옵션이다.

1.       예) www.cnn.com/ASIA -D 1 -X /ASIA/Japan

2.       ASIA내용 중 Japan내용은 제외하고자 하는 것.

l          실행 주의 사항

Ø         www.cnn.com/ASIA/ 에서는 ASIA를 디렉토리로 인식하여 D=1 이상인 경우 하위 링크를 따라간다.

Ø         www.cnn.com/ASIA 에서는 ASIA를 화일로 인식하여 하위 링크를 따라가지 않는다.
